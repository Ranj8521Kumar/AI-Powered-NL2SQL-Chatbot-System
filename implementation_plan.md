# AI-Powered NL2SQL System — Implementation Plan

## Overview

Build a complete, production-quality **Natural Language to SQL (NL2SQL)** system using **Vanna 2.0** and **FastAPI**, deployed over a **SQLite clinic database**. The system accepts natural language questions, generates and validates SQL, executes it, and returns structured results with Plotly charts and natural language summaries.

This plan strictly follows Vanna 2.0's agent-based architecture (NOT the deprecated 0.x `vn.train()` / ChromaDB pattern).

---

## Architecture Summary (from provided diagram)

```
USER LAYER          → Chat UI / Web Interface
      ↓  HTTP Request
API LAYER           → FastAPI Backend (POST /chat, GET /health)
      ↓  Validated Request
AI AGENT LAYER      → Vanna 2.0 Agent
                        ├─ LLM (Gemini 2.5-flash)
                        ├─ ToolRegistry (RunSqlTool, VisualizeDataTool, SaveQ&ATool, SearchSavedTool)
                        └─ DemoAgentMemory (learned Q&A pairs)
      ↓  NL→SQL Generation
PROCESSING LAYER    → SQL Validator (SELECT only, block dangerous keywords)
      ↓  Validated SELECT
DATA LAYER          → SqliteRunner → clinic.db
                        (appointments, patients, doctors, invoices, treatments)
      ↓  Results
OUTPUT LAYER        → rows/columns + NL Summary + Plotly Visualization
      ↓  Formatted Response
USER LAYER          ← JSON Response
```

### Step 4 — Vanna 2.0 Agent

#### [NEW] `project/vanna_setup.py`
Uses **correct Vanna 2.0 import paths**:
```python
from vanna import Agent, AgentConfig
from vanna.servers.fastapi import VannaFastAPIServer
from vanna.core.registry import ToolRegistry
from vanna.core.user import CookieEmailUserResolver
from vanna.integrations.gemini import GeminiLlmService
from vanna.tools import RunSqlTool, VisualizeDataTool
from vanna.integrations.sqlite import SqliteRunner
from vanna.integrations.local.agent_memory import DemoAgentMemory
```
- Creates `GeminiLlmService` with `gemini-2.5-flash` model
- Creates `ToolRegistry` with `RunSqlTool`, `VisualizeDataTool`
- Creates `DemoAgentMemory(max_items=1000)`
- Creates `CookieEmailUserResolver` for user identification
- Returns configured `Agent` instance via `get_agent()` factory function

### Step 5 — Memory Seeding

#### [NEW] `project/seed_memory.py`
Pre-seeds **15 Q&A pairs** across all categories using `DemoAgentMemory`:
- **Patient queries** (4): count, list by city, filter by gender, repeat visitors
- **Doctor queries** (3): appointments per doctor, busiest doctor, by specialization
- **Appointment queries** (3): by status, by month, by doctor
- **Financial queries** (3): total revenue, unpaid invoices, avg cost
- **Time-based queries** (2): last 3 months, monthly trends

---

### Step 6, 7, 8, 9 — FastAPI Backend

#### [NEW] `project/main.py`
Custom FastAPI app wrapping the Vanna Agent (Option B for full control):
- `POST /chat` — accepts `{"question": "..."}`, runs agent, returns structured JSON
- `GET /health` — returns status, DB connectivity, memory item count
- Includes CORS middleware for frontend
- Integrates SQL validator before execution
- Integrates chart generator for aggregation results

#### [NEW] `project/sql_validator.py`
- Blocks: `INSERT`, `UPDATE`, `DELETE`, `DROP`, `ALTER`, `EXEC`, `GRANT`, `REVOKE`, `SHUTDOWN`, `xp_`, `sp_`
- Blocks system table access: `sqlite_master`, `sqlite_schema`
- Only allows `SELECT` queries

#### [NEW] `project/chart_generator.py`
- Detects numeric aggregation columns
- Auto-generates Plotly bar/line/pie charts
- Returns Plotly JSON dict (serializable)
